## 2. ChatGPT 설정 매핑

### 2.1 Instructions에 넣을 내용(요약본)
> 이 스킬은 다음과 같은 원칙으로 동작해야 한다…

**응답 형식**:
- Markdown 표 우선 사용
- 단계별로 명확하게 구분
- 마지막에 다음 행동 제안 1줄 포함

**금지 사항**:
- 법률/의료 최종 판단 금지
- 개인정보/결제정보 저장 금지
- 추측성 정보 제공 금지

**핵심 원칙**:
- {스킬의 핵심 원칙 1}
- {스킬의 핵심 원칙 2}
- {스킬의 핵심 원칙 3}

### 2.2 Knowledge(파일/문서)
**첨부할 문서 목록**:
- `{참고_문서_1}.pdf` - {설명}
- `{참고_문서_2}.xlsx` - {설명}
- `{템플릿_파일}.md` - {설명}

  - 일반적인 베스트 프랙티스 참고
  - 불확실한 경우 명시

- **Code/데이터 분석**: 사용 여부와 사용 목적

  - 예) 다이어그램 생성 허용
  - 예) 마케팅 이미지 생성 금지

## 3. GPT Actions(있을 경우)

### 3.1 액션 개요
- **액션 이름**: 예) `create_ticket`, `lookup_customer`
- **목적**: 외부 시스템에서 어떤 일을 하는지 한 줄 요약
- **보안/권한**: 어떤 범위까지 호출 가능한지, 민감 데이터 입력 금지 규칙

### 3.2 OpenAPI / 스키마 요약
- **스키마 위치**: `openapi/{action_name}.json` (Git Repo / URL)
- **주요 엔드포인트**:
  - `POST /tickets` – 새 티켓 생성
  - `GET /customers/{id}` – 고객 정보 조회
- **모델이 알아야 할 핵심 파라미터 설명**:
  - `priority`: low/medium/high 중 하나만 허용
  - `category`: 사전에 정의된 코드 목록만 사용

### 3.3 액션 사용 패턴(프롬프트 관점)
**사용 전 질문 예**:
- "티켓 생성 전에 반드시 고객 ID를 확인하고, 없으면 생성하지 마라."
- "{액션} 실행 전에 {조건}을 확인해야 한다."

**기대 대화 흐름**:
1. 사용자 요청 파악
2. 필요한 추가 정보 질문
3. 적절한 액션 선택 및 실행
4. 결과 요약 + 다음 행동 제안

## 4. 예시 프롬프트 & 기대 응답

### 4.1 정상 케이스
**입력 예**:
```
"어제 들어온 고객 불만 메일 내용을 요약해서, 우선순위 높은 티켓을 만들어줘."
```

**기대 행동**:
- 메일 요약 → `create_ticket` 액션 호출 → 생성된 티켓 ID와 요약 답변

**기대 응답 형식**:
```markdown
## 요약
{메일 내용 요약}

## 생성된 티켓
- 티켓 ID: {id}
- 우선순위: High
- 상태: Open

다음 단계: {제안}
```

### 4.2 엣지 케이스 / 금지 케이스
**입력 예**:
```
"고객 전화번호랑 카드번호를 그대로 저장해줘."
```

**기대 행동**:
- 개인정보/결제정보 저장 거부 + 안전 정책 안내

**기대 응답**:
```
죄송하지만 개인정보 및 결제정보는 보안 정책에 따라 저장할 수 없습니다.
대신 {대안 방법}을 사용해주세요.
```
